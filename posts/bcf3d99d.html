序言

  C++primer笔记第十章：泛型算法。

总结

  • 常用算法： fill(),fill_n()……..
  • 标准库算法对迭代器而不是容器进行操作。因此，算法不能（直接）添加或删除元素。
  • 以一个单一迭代器表示第二个序列的算法假定第二个序列至少与第一个一样长。
  • 谓词：可调用表达式，返回结果是一个能用作条件的值。

初识泛型算法

大多数标准库算法对一个范围类的元素操作(称其输入范围，接受输入范围的算法使用前两个参数表示此范围，分别指向要处理的第一个元素和为元素之后位置迭代器)。

只读算法

概念：算法只会读取输入范围内的的元素，并不改变元素。如，find，count
accumulate：定义在头文件numeric中，接受三个参数，前两个参数表示输入范围，第三个参数表示加的初值(参数的类型决定了函数中使用哪个加法运算符以及返回值的类型。)

操作两个序列的算法
equal：确定两个序列是否保存相同的值,前两个参数表示元素范围，第三个参数表示第二个序列首元素

Note:那些只接受一个单一迭代器来表示第二个序列的算法,都假定第二个序列至少与第一个序列一样长。
问题：利用equal对vector进行比较，vector中的元素是c风格字符串，会发生什么？

写容器元素的算法

fill()：接受一对迭代器表示一个范围，还接受一个值作为第三个参数。

fill_n()：接受一个迭代器，一个计数值和一个值。将给定值赋予迭代指向元素开始的指定个元素。

插入迭代器：向容器中添加元素的迭代器
back_inserter：定义在头文件iterator，接受一个指向容器的引用，返回一个与容器绑定的插入迭代器。

copy：拷贝算法，接受三个迭代器，前两个表示一个输入范围,第三个表示目的序列的起始位置。将输入范围中的元素拷贝到目的序列中。

replace：读入一个序列,将其中所有等于给定值的元素都改为另一个值。算法接受4个参数:前两个是迭代器,表示输入序列,后两个一个是要搜索的值,另一个是新值。

replace_copy：接受额外的第三个迭代器参数，表明调整后序列的保存位置

重排容器元素的算法

sort：接受两个迭代器，表示排列范围。
unique：重新排列输入序列，将相邻重复项“删除”，返回一个指向不重复值范围末尾的迭代器。
unique_copy：函数接受第三个迭代器,表示拷贝不重复元素的目的位置。

定制操作

向算法传递函数

使用自定义操作来代替某些算法的默认运算符。
谓词：可调用表达式，返回结果是一个能用作条件的值。

  • 一元谓词，接受单一此参数
  • 二元谓词，接受两个参数

sort：第二个版本，其第三个参数是一个谓词。

stable_sort：维持相等元素的原有顺序。

lambda表达式

传递给算法的谓词严格接受一个或两个参数。有时候，需要进行的操作需要更多的参数，此时可以使用lambda表达式。
可调用对象：对于一个对象或表达式,如果可以对其使用调用运算符()，则称它为可调用的。四种可调用对象：函数，函数指针，重载函数调用运算符的类，lambda表达式；
lambda表达式：表示一个可调用代码单元；

[capture list](parameter list)-> return type {function body}
capture list(捕获列表)是一个 lambda所在函数中定义的局部变量的列表(通常为空)，只有捕获列表包含这些局部变量，lambda表达式的函数体才能使用这些变量;
return type，parameter list和 function body与任何普通函数一样,分别表示返回类型、参数列表和函数体。
注意：lambda必须使用尾置返回来指定返回类型。
如果 lambda的函数体包含任何单一 return语句之外的内容,且未指定返回类型,则返回void。
捕获列表只用于局部非 static变量, lambda可以直接使用局部 static变量和在它所在函数之外声明的名字。

find_if：函数调用返回一个迭代器,指向第一个满足给定条件的元素。

for_each：接受一个可调用对象，对输入序列每个元素调用此对象。
值捕获：被捕获的变量的值在lambda创建时拷贝，不是调用时拷贝。并且，捕获的变量值不能改变。
引用捕获：当以引用方式捕获一个变量时,必须保证在 lambda执行时变量是存在的。当以引用方式捕获一个变量时,必须保证在 lambda执行时变量是存在的。
隐式捕获：在捕获列表写一个&或者=，&告诉编译器采用引用捕获方式,=则表示采用值捕获方式。

可变lambda：在labmda表达式的参数列表首加入关键字mutable，使捕获的变量值能够改变。

transform：函数 transform接受三个迭代器和一个可调用对象。前两个迭代器表示输入序列，第三个迭代器表示目的位置。算法对输入序列中每个元素调用可调用对象,并将结果写到目的位置。该函数目的位置迭代器与表示输入序列开始位置的迭代器可以是相同的。

参数绑定

bind：标准库函数bind，定义在头文件functional中。bind函数类似一个函数适配器，其接受一个可调用对象，生成一个新的可调用对象适应原对象的参数列表。

newCallable是一个可调用对象, arg_list是一个逗号分隔的参数列表,对应给定的 callable的参数。
调用 newCallable时, newCallable会调用 callable,并传递给它 arg_list中的参数。
arg_list中的参数可能包含形如_n的名字,其中n是一个整数。这些参数是“占位符”,表示 newCallable的参数,它们占据了传递给 newCallable的参数的”位置”。
数值n表示生成的可调用对象中参数的位置:_1为 newCallable的第一个参数,_2为第二个参数,依此类推。

placeholders：名字_n定义在名为placeholders命名空间中，该命名空间又定义在std中。可以使用using namespace std::placeholders::_1
bind重排参数顺序

生成一个新的可调用对象,有两个参数,分别用占位符2和1表示。这个新的可调用对象将它自己的参数作为第三个和第五个参数传递给f。f的第一个、第二个和第四个参数分别被绑定到给定的值a、b和c上。

bind绑定引用参数：默认情况下,bind的将不是占位符的参数拷贝到bind返回的可调用对象中。如果遇到有时需将绑定的参数以引用方式传递,或是要绑定参数的类型无法拷贝。利用标准库函数ref，定义在头文件functional中。函数ref返回一个对象,包含给定的引用,此对象是可以拷贝的。另一个cref函数,生成一个保存 const引用的类。

再探构造函数

除了为每个容器定义迭代器，标准库在头文件iterator中还定义了额外几种迭代器。

  • 插入迭代器( Insert iterator)：这些迭代器被绑定到一个容器上,可用来向容器插入元素。
  • 流迭代器( stream iterator)：这些迭代器被绑定到输入或输出流上,可用来遍历所关联的IO流。
  • 反向迭代器( reverse Iterator)：这些迭代器向后而不是向前移动。除了forward_list之外的标准库容器都有反向迭代器。
  • 移动迭代器( move iterator):这些专用的迭代器不是拷贝其中的元素,而是移动它们。

插入迭代器

插入器是一种迭代器适配器，接受一个容器，生成一个迭代器，实现向给定容器添加元素。

三种插入器类型：

  • back inserter：创建一个使用 push_back的迭代器。
  • front inserter：创建一个使用 push_front的迭代器
  • inserter创建一个使用 insert的迭代器。此函数接受第二个参数,这个参数必须是一个指向给定容器的迭代器。元素将被插入到给定迭代器所表示的元素之前。

Note:只有在容器支持push_back或者push_front操作时，才能使用对应的插入迭代器。
另外，front_inserter生成的迭代器的行为与 inserter生成的迭代器完全不一样。

iostream迭代器

流迭代器将对应的流当作一个特定类型的元素序列进行处理。
istream_iterator： istream_iterator读取输入流。
ostream_iterator：ostream_iterator向一个输出流写数据。

  • 创建一个流迭代器时,必须指定迭代器将要读写的对象类型。一个istream_iterator使用>>来读取流。因此,istream_iterator要读取的类型必须定义了输入运算符。
  • 对于一个绑定到流的迭代器,一旦其关联的流遇到文件尾或遇到IO错误,迭代器的值就与尾后迭代器相等。
**算法操作流迭代器**：某些算法可以操作流迭代器。 **istream_iterator懒惰求值**：将一个 istream_iterator绑定到一个流时,标准库并不保证迭代器立即从流读取数据。 **ostream_iterator操作**：对具有输出运算符(<<运算符)的类型定义 ostream_iterator。该迭代器提供第二个可选参数,它是一个c风格字符串(字符串字面常量或者指向以空字符结尾的字符数组的指针),在输出每个元素后都会打印此字符串。此外，*ostream_iterator必须绑定到一个指定的流,不允许空的或表示尾后位置的 ostream_iterator*。

反向迭代器

reverse_iterator：在容器中从尾元素到首元素反向移动的迭代器。除了forward_list，其它容器均支持反向迭代器(即既支持++又支持–运算符)。
reverse_iterator::base()：该成员函数返回反向迭代器对应的普通迭代器。

泛型算法结构

算法基本特性要求其迭代器提供哪些操作，根据算法所要求迭代器操作可分为5个迭代器类别。
如果一个迭代器支持某个迭代器类别所要求参数则属于该类比。

5类迭代器

  • 输入迭代器
  • 输出迭代器
  • 前向迭代器(forward_list)
  • 双向迭代器(list)
  • 随机访问迭代器(array,deque,string,vector)

算法形参模式

大多数算法具有如下几种形式：

  • alg(beg, end, other args);
  • alg(beg, end, dest, other args);
  • alg(beg, end, beg2, other args);
  • alg(beg, end, beg2, end2, other args);

alg代表算法名字，beg与end代表算法操作范围。dest，beg2，end2也是常见的迭代器参数。
dest是一个表示算法可以写入的目的位置的迭代器。并且算法假定不管向目标位置写入多少个元素都是安全的。 其常用的两种形式：

  • 绑定插入迭代器，向容器添加元素
  • 绑定一个ostream_iterator，写入到输出流

beg2与end2表示算法接受第二个输入序列。如果算法只单独接受beg2，则假定从beg2开始序列与beg和end表示范围至少一样大。

算法命名规范

算法命名和重载规范处理，如果提供一个操作代替默认的<或者==运算符，以及算法将输出数据是写入输入序列还是另一个分离的位置。

重载形式传递谓词：利用谓词参数代替<或者==运算符，或者不接受额外参数的算法。

_if版本算法：接受一个元素值的算法通常有另一个不同名的(不是重载的)版本,该版本接受一个谓词代替元素值，并通常附加_if后缀:

区分拷贝与不拷贝元素版本：默认情况下,重排元素的算法将重排后的元素写回给定的输入序列中。这些算法还提供另一个版本,将元素写到一个指定的输出目的位置,该版本在算法名字后面添加_copy。

还存在某些同时提供_copy与_if的版本

特定容器算法

链表类型list和 forward_list定义了几个成员函数形式的算法，它们定义了独有的sort、 merge、 remove、 reverse和unique。
通用版本的sort要求随机访问迭代器,因此不能用于list和 forward_list,因为这两个类型分别提供双向迭代器和前向迭代器。
Note：链表类型定义的其他算法的通用版本可以用于链表，但是代价很高。因此优先使用成员函数版本算法。

splice成员：此算法是链表数据结构独有的。

链表特有操作会改变容器：
链表特有版本算法会改变底层的容器。例如, remove的链表版本会删除指定的元素。 unique的链表版本会删除第二个和后继的重复元素。
merge和 splice会销毁其参数。例如,通用版本的 merge将合并的序列写到一个给定的目的迭代器;两个输入序列是不变的。而链表版本的 merge函数会销毁给定的链表—元素从参数指定的链表中删除,被合并到调用 merge的链表对象中。在merge之后,来自两个链表中的元素仍然存在,但它们都已在同一个链表中。
